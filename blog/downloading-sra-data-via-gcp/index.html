Downloading SRA data via GCP

Tim Stuart 2021-06-14 2 minute read

The SRA now stores the data in the originally uploaded format for many SRA accessions. This is particularly useful for single-cell data, as the conversion to SRA FASTQ often strips cell barcodes from the file, as read names and read tags are not preserved. SRA enables you to download the original files provided by the authors, provided you pay for the download cost. They provide download links for AWS and Google Cloud Compute. I found that many of the AWS links are in “cold-storage”, which doesn’t allow immediate download, and you need to make a request and wait for the data to be moved somewhere where it can be accessed. Google cloud seems to have more data immediately available.

This process takes some wrangling if you’re not familiar with how to use the AWS or GCP command line tools to download data from cloud storage, or how to specify the “requester-pays” option. Here I’ll give a very quick walkthrough of how to use GCP to download data from the SRA:

1. Set up Google cloud SDK

The Google cloud SDK is the command-line toolkit used for accessing Google cloud. You can install it from the Google cloud website, or from conda:

mamba install -c conda-forge google-cloud-sdk

Next, you need to configure gcloud to access your account.

gcloud init

2. Download data

Data can be downloaded using the gsutils cp command. Files that require the requester pays for egress will also need a project specified, so that your project gets billed for the download. For example:

gsutil -u proj cp gs://sra-pub-src-18/SRR10343083/scATAC_CD34_D7T1.bam.1 .

Note that you’ll need to set up a project to use on your Google cloud account.

This should download the specified file. In this case, the BAM file is ~40 Gb and I was charged ~$6 including archive data retrieval and download.
